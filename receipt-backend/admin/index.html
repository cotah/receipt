How it works

1. Fetches all products from collective_prices

2. Generates new sorted product keys (order-independent: "Tesco Chicken" = "Chicken Tesco")

3. Finds entries with same key + same store = confirmed duplicate

4. Keeps the cheapest price, removes extras

5. Updates barcode_catalog keys for consistency

No AI/LLM used — pure deterministic string comparison. Zero cost, zero hallucination risk.
