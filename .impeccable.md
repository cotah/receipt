# SmartDocket — Impeccable Design Context

This file is the single source of truth for design decisions in SmartDocket. Any design or UI work should start by reading this file.

---

## Project Snapshot

- **Product**: SmartDocket — a grocery price-comparison app for Ireland.
- **Platform**: React Native + Expo (iOS + Android). `userInterfaceStyle: "dark"`.
- **Core job**: help shoppers compare prices across **Lidl, Aldi, Tesco, Dunnes Stores** (and SuperValu) and decide where to shop this week.
- **Bundle id / domain**: `ie.receipt.app` / `smartdocket.ie`.

## Design Context

### Users

Two primary personas, both price-sensitive grocery shoppers in Ireland:

1. **Brazilian shoppers in Ireland** — often newcomers, navigating unfamiliar supermarket brands, comparing prices in EUR against a mental BRL baseline. Likely to appreciate clear Portuguese-friendly copy, low-friction onboarding, and reassurance that the app is honest about prices.
2. **Irish shoppers** — locals who already know the big 4 chains but want a faster way to see where this week's basket is cheapest. Value accuracy, speed, and local cues (store brand colors, familiar product names, Irish supermarket iconography).

**Context of use**: phone in hand, often mid-shop in a supermarket aisle, sometimes planning at home on the sofa. Frequently one-handed. Network may be poor inside stores.

**Jobs to be done**:
- Scan a receipt and see where each item would have been cheaper.
- Build a shopping list and check the total across the 4 chains.
- Spot price changes / alerts on recurring items.
- Earn points / rewards for contributing prices.

### Brand Personality

Three words: **modern, trustworthy, budget-friendly**.

Supporting tone words: **honest, local, calm, confident, easy**.

Emotional goal: the user should feel **in control of their grocery spend** — not anxious, not gamified into exhaustion, not patronised. The app is a quiet advantage, not a loud coupon site.

**Anti-references** (what this explicitly should NOT feel like):
- Flashy cashback / coupon apps (neon gradients, confetti everywhere, constant urgency).
- Generic fintech dashboards (cyan-on-dark, purple-to-blue gradients, sparkline decoration).
- Budget-supermarket aesthetics that feel cheap (harsh yellows, shouty red price tags).

### Aesthetic Direction

**Theme**: dark mode, permanently. Rationale: the app is used in supermarket aisles under bright overhead lighting where a dark UI reduces glare, and at home in the evening when planning a shop. The dark green base also distances SmartDocket from both generic "fintech dark" and bright promo-app styling.

**Visual tone**: *refined forest — warm, calm, slightly editorial*. Deep evergreen canvas, translucent "glass" surfaces, mint-green as the savings signal, warm amber as the rewards / highlight accent. Store colors are used as *muted tints* so Tesco/Lidl/Aldi/Dunnes/SuperValu are identifiable at a glance without the UI becoming a logo wall.

**Typography** (already installed as design tokens in `receipt-mobile/constants/typography.ts`):
- Display: **DM Serif Display** — used sparingly for hero numbers, empty-state headlines, and section titles that need editorial weight.
- Body: **DM Sans** (regular / medium / semibold / bold).
- Numerics / prices: **JetBrains Mono** (medium / semibold / bold) — gives prices a tabular, trustworthy feel and keeps euro amounts aligned.

> Note on the reflex-fonts list: DM Sans and DM Serif Display normally count as "AI-default" choices. In this project they are already shipped in production tokens and load via `expo-font`. Do **not** swap them out during design work — change them only as an intentional brand project with the user. New projects inside this repo should still pick non-default fonts.

**Color palette** — match the tokens in `receipt-mobile/constants/colors.ts`:

- **Canvas**: `#0d2818` (deep forest) → `#0a1f14` (near-black green).
- **Primary brand**: `#1B5E3B` (Irish green), hover/active `#7DDFAA` (mint).
- **Savings accent**: mint `#7DDFAA` with a soft halo `rgba(80,200,120,0.12-0.15)`.
- **Rewards / streaks accent**: amber `#F0D68A` with soft `rgba(212,168,67,0.10-0.12)`.
- **Warning / price-up**: `#F09595` (muted coral, never fire-engine red).
- **Info**: `#85B7EB` (muted blue).
- **Surfaces**: translucent white glass at 4–12% opacity over the canvas. Borders at 12–20% white.
- **Text**: white at 100 / 55 / 35% for primary / secondary / tertiary. Inverse text uses the canvas color.
- **Store tints** (use for chips, badges, and filter pills — not full backgrounds):
  - Tesco `rgba(133,183,235,0.40)`
  - Lidl `rgba(240,153,123,0.40)`
  - Aldi `rgba(124,140,240,0.45)`
  - Dunnes `rgba(93,202,165,0.35)`
  - SuperValu `rgba(240,214,138,0.35)`

**Spacing & radius**: 4-pt scale (`xs 4 / sm 8 / md 16 / lg 24 / xl 32 / xxl 48`). Radii 10 / 14 / 18 / 24 / full. Prefer `gap` over margin. Cards use radius `lg` (18); modals and hero surfaces use `xl` (24).

**Motion**: exponential ease-out for entrances (300–500 ms). Prices and totals use a brief number-count animation when they change. Haptics (`expo-haptics`) on confirmations, never on idle scroll. Respect reduced-motion.

### Design Principles

1. **Price is the hero.** Numbers are the most important element on any screen. Use JetBrains Mono, give prices extra visual weight, and never bury the cheapest option behind decoration.
2. **Calm, not cheap.** Dark forest + mint + amber reads as *considered and honest*. Avoid neon, hard red, confetti, and "SAVE NOW" urgency. Savings are stated, not shouted.
3. **Local cues earn their place.** Store colors, the euro sign, and Irish supermarket semantics should appear where they help the shopper decide — not as decoration.
4. **Built for the aisle.** One-handed reach, thumb-sized targets (min 44 pt), clear contrast under store lighting, and fast feedback even on weak signal.
5. **Bilingual by design.** Layouts must survive Portuguese copy that is ~15–25% longer than English. Never lock widths to English strings; use `flexShrink`, wrapping, and truncation thoughtfully.
6. **Trust through transparency.** Show where a price came from (receipt, user contribution, store), when it was last seen, and when you are uncertain. Never fake precision.

## Working Agreements (for future design work in this repo)

- Use the existing tokens in `constants/colors.ts` and `constants/typography.ts`. Extend them rather than hard-coding new hex / font values in components.
- When introducing a new surface style, check whether an existing glass variant already fits.
- When picking a color for a new concept, start from the accent family (green = savings, amber = rewards, coral = warning, blue = info) before reaching for a new hue.
- When displaying a store, use its tint from `Colors.stores.*` — not the store's official brand hex.
- Headlines over ~18 pt should use DM Serif Display. Everything else defaults to DM Sans. Prices / totals / quantities use JetBrains Mono.
